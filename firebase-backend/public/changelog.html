What changed

Changelog

Every update to ZAS Safeguard, in order.

v1.2.0December 2025
Security Hardening & Platform Stability
Security
  • Added token authentication to registerDevice and updateDeviceStatus endpoints — previously unauthenticated
  • Removed eval() from URL scanner, replaced with safe JSON.parse pattern
  • Fixed CORS configuration to allow extension service worker requests in logBlockEventHttp
  • Added brute-force limit (5 attempts) to owner mode unlock flow
  • Added SMS rate limiting to phone verification — 5 sends per hour per user
  • Added explicit Firestore security rules for 8 previously unprotected server-only collections
  • Replaced navigator.sendBeacon with fetch + keepalive to support Authorization headers
  • Added sanitization to AI prompt inputs to prevent injection attacks
  • Capped past_due subscription grace period at 48 hours
Bug Fixes
  • Fixed monthly plan not loading in checkout — createSubscriptionIntent now reads price IDs from Firestore config
  • Fixed free trial being available unlimited times — trial_used is now correctly stamped on the device registry after first trial
  • Fixed AI content analysis silently failing for all paid users — was reading subscription.status instead of subscription.plan_status
  • Fixed custom blocklist domain validation — added regex with wildcard support
  • Fixed logSecurityEventHttp auto-creating phantom device documents for any authenticated user
  • Fixed full URL with query parameters being logged to Firestore — now logs origin and pathname only
Dashboard
  • Fixed Trial Expired modal showing for brand-new users who never had a trial
  • Fixed ads blocked counter undercounting — now reads count field from batched log documents
  • Fixed trial days remaining always showing null — Firestore field name corrected (trial_end)
  • Fixed new users seeing Expired badge in sidebar
  • Fixed extension stats being fetched but never displayed — dashboard now shows live count from extension
  • Fixed alerts showing in wrong order — now sorted by createdAt field
  • Fixed protection status always showing Active regardless of extension state
  • Replaced all native alert() popups in Settings with inline toast notifications
  • Fixed status card appearing below expired section — now always renders first
  • Fixed duplicate expiry message appearing on dashboard
  • Fixed motivational quote card not showing for expired users
New
  • YouTube ad blocking now correctly disabled when subscription expires — no more black screen
  • Firebase ID token auto-refreshes every 45 minutes — eliminates silent auth failures after 1 hour
  • Extension ID is now discovered automatically via announcement message — no hardcoded IDs
