Changelog

Every update to ZAS Safeguard, most recent first.

v1.2.0December 2025
Security Hardening & Platform Stability
Security
  • Added token authentication to registerDevice and updateDeviceStatus — previously completely unauthenticated
  • Removed eval() from URL scanner — converted urlPatterns.js to safe JSON.parse pattern
  • Fixed CORS to allow browser extension requests in logBlockEventHttp
  • Added 5-attempt brute-force limit to owner mode PIN unlock
  • Added SMS rate limiting to phone verification — capped at 5 per hour per user
  • Added Firestore security rules for 8 previously unprotected server-only collections
  • Replaced sendBeacon with fetch+keepalive to properly attach Authorization headers
  • Added AI prompt input sanitization to prevent prompt injection
  • Capped past_due subscription grace period at 48 hours
Bug Fixes
  • Fixed monthly plan not loading at checkout — createSubscriptionIntent now reads price IDs from Firestore config
  • Fixed free trial being available unlimited times — trial_used flag now stamped on device registry after first trial
  • Fixed AI content analysis failing silently for paid users — was reading wrong subscription field
  • Fixed custom blocklist domain validation to support wildcard rules
  • Fixed logSecurityEventHttp auto-creating phantom device documents
  • Fixed full URL including query params being logged — now logs origin and pathname only
Dashboard
  • Fixed Trial Expired modal showing for brand-new users who never had a trial
  • Fixed ads blocked counter — now reads count field from batched log documents correctly
  • Fixed trial days countdown always showing null — Firestore field name corrected
  • Fixed new users seeing Expired badge in sidebar
  • Dashboard now shows live extension ad-block count instead of stale Firestore stat
  • Fixed alerts showing in wrong order — sorted by createdAt correctly
  • Fixed protection status always showing Active regardless of extension state
  • Replaced all native browser alert() popups in Settings with inline toasts
  • Fixed status card rendering below expired section — now always renders first
  • Fixed duplicate expiry message on dashboard
  • Fixed motivational quote card not showing for expired users
New
  • YouTube ad blocking correctly disabled when subscription expires — no more black screen
  • Firebase ID token now auto-refreshes every 45 minutes — eliminates silent auth failures after 1 hour
  • Extension ID discovered automatically via announcement message — no hardcoded IDs in repo
