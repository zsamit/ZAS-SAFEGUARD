Security

Last updated: December 2025

How ZAS Safeguard protects your data, what the extension can and cannot access, and how to report a vulnerability.

Responsible disclosure

If you discover a security vulnerability in ZAS Safeguard, report it to us privately before public disclosure. We take all reports seriously and respond within 48 hours.

Report a vulnerability

Email: info@zasgloballlc.com

Include: description of the vulnerability, steps to reproduce, and potential impact. We acknowledge within 48 hours and resolve critical issues within 7 days. We will credit you in our changelog with your permission.

Data security

  • All traffic encrypted in transit via TLS 1.2+
  • Authentication: Firebase Auth — industry-standard JWT tokens, 1-hour expiry with auto-refresh
  • Database: Firestore with field-level security rules — users can only access their own data
  • Payments: Stripe — we never store card numbers
  • Passwords: Firebase Auth — we never store plaintext passwords
  • PIN codes: hashed with PBKDF2 + SHA-512 with random salt

Extension permissions explained

PermissionWhy it is needed
webNavigationDetects navigation to new pages so the extension can check them against blocklists in real time
webRequestIntercepts and analyses network requests for ad and tracker blocking
declarativeNetRequestMV3 API for blocking requests using local rules — more private than webRequest
storageStores your auth token, settings, and stats locally on your device
tabsReads the current tab URL to determine whether blocking rules apply
alarmsRuns the token refresh alarm every 45 minutes to keep you logged in
identityUsed for Google sign-in

What the extension cannot do

  • Cannot read emails, messages, or documents
  • Cannot take screenshots
  • Cannot access files on your device
  • Cannot run when the browser is closed
  • Cannot access Incognito tabs unless you explicitly enable it

Reporting abuse

If you believe ZAS Safeguard is being misused: info@zasgloballlc.com
