Security

Last updated: December 2025

This page describes how ZAS Safeguard approaches security — how we protect your data, what the extension can access and why, and how to report a vulnerability.

Responsible disclosure

If you discover a security vulnerability in ZAS Safeguard, we ask that you report it to us privately before disclosing it publicly. We take all reports seriously and will respond within 48 hours.

Report a vulnerability

Email: security@zassafeguard.com

Please include: a description of the vulnerability, steps to reproduce, and the potential impact. We will acknowledge your report within 48 hours and aim to resolve critical issues within 7 days.

What happens after you report

  • We acknowledge your report within 48 hours
  • We assess the severity and assign a priority
  • We work with you to understand and reproduce the issue
  • We fix the vulnerability and deploy the fix
  • We notify you when the fix is live
  • With your permission, we will credit you in our changelog

Data security

All communication between the extension, the dashboard, and our backend is encrypted in transit via TLS 1.2+. User data is stored in Google Firebase, which provides encryption at rest.

  • Authentication: Firebase Authentication — industry-standard JWT tokens, 1-hour expiry with automatic refresh
  • Database: Firestore with field-level security rules — users can only access their own data
  • Payments: Stripe — we never store card numbers. Stripe handles all payment data.
  • Passwords: Firebase Authentication — we never store plaintext passwords
  • PIN codes: hashed with PBKDF2 + SHA-512 with random salt — never stored in plaintext

Extension permissions explained

ZAS Safeguard requests the following permissions in manifest.json. Here is exactly why each one is needed:

PermissionWhy it is needed
webNavigationDetects when you navigate to a new page so the extension can check it against blocklists in real time
webRequestRequired to intercept and analyse network requests for ad blocking and tracker removal
declarativeNetRequestThe MV3 API for blocking network requests using predefined rules — more private than webRequest because the rules run locally without seeing request data
storageStores your auth token, extension settings, and ad block stats locally on your device
tabsAllows the extension to read the current tab URL to determine whether blocking rules apply
alarmsPowers the token refresh alarm (runs every 45 minutes to keep you logged in)
identityUsed for Google sign-in authentication

What the extension cannot do

  • It cannot read the content of your emails, messages, or documents
  • It cannot take screenshots
  • It cannot access files on your computer
  • It cannot run when the browser is closed
  • It cannot access sites you visit in Incognito mode unless you explicitly enable it

Reporting abuse

If you believe someone is using ZAS Safeguard in a way that violates our terms or harms others: security@zassafeguard.com
